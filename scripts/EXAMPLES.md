# Example MCP Server Configurations

This directory contains example configuration files generated by the installation scripts.

## VSCode/Cursor/Windsurf Configuration Example

File: `settings.json`

```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "${workspaceFolder}"]
    },
    "git": {
      "command": "uvx",
      "args": ["mcp-server-git", "--repository", "${workspaceFolder}"]
    },
    "fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    },
    "everything": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-everything"]
    },
    "sequentialthinking": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sequentialthinking"]
    },
    "time": {
      "command": "uvx",
      "args": ["mcp-server-time"]
    }
  }
}
```

## Adding Custom Servers

You can manually add additional MCP servers to the configuration. For example:

### PostgreSQL Server

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://localhost/mydb"]
    }
  }
}
```

### Brave Search Server

```json
{
  "mcpServers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

### Slack Server

```json
{
  "mcpServers": {
    "slack": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-your-token-here",
        "SLACK_TEAM_ID": "T01234567"
      }
    }
  }
}
```

## Configuration Tips

### Using Workspace Variables

For VSCode/Cursor/Windsurf, you can use the `${workspaceFolder}` variable:

```json
{
  "filesystem": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "${workspaceFolder}/src"]
  }
}
```

### Adding Multiple File System Paths

For multiple allowed directories:

```json
{
  "filesystem": {
    "command": "npx",
    "args": [
      "-y",
      "@modelcontextprotocol/server-filesystem",
      "/path/to/dir1",
      "/path/to/dir2",
      "/path/to/dir3"
    ]
  }
}
```

### Environment Variables

Always use environment variables for sensitive data:

```json
{
  "server-name": {
    "command": "command",
    "args": ["args"],
    "env": {
      "API_KEY": "your-key-here",
      "API_SECRET": "your-secret-here"
    }
  }
}
```

### Using Python Virtual Environments

If you prefer pip over uvx:

```json
{
  "git": {
    "command": "python",
    "args": ["-m", "mcp_server_git"]
  }
}
```

## Verifying Configuration

After configuration, verify the setup:

1. **Restart your IDE completely**
2. **Check MCP server status** in your IDE's MCP panel/extension
3. **Test a command** - Ask your AI assistant to use a specific MCP tool

Example test commands:
- "List the files in this directory" (filesystem)
- "What's the git status of this repository?" (git)
- "Remember that I prefer using Python for data analysis" (memory)
- "What's the current time in London?" (time)

## Troubleshooting

### Server Not Starting

Check if you can run the server manually:

```bash
# Test TypeScript server
npx -y @modelcontextprotocol/server-memory

# Test Python server
uvx mcp-server-git
```

### Permission Errors

On Unix systems, ensure the scripts are executable:

```bash
chmod +x scripts/install_mcp_servers.sh
chmod +x scripts/install_mcp_servers.py
```

### Path Issues

Use absolute paths if relative paths cause issues:

```json
{
  "filesystem": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/absolute/path/to/directory"]
  }
}
```
